iOS开发对(null)/< null >的处理

发表于 2016-01-21 | 分类于 遇到的trouble |

在开发过程中遇到过后台返回的数据为<null>

在开发过程中有价格数据要显示在界面上，于是就做了处理

self.PriceText.text = [NSString stringWithFormat:@" %.2f",getOrder.price.floatValue];

结果程序就崩溃了。为了查出原因就看了下数据，后台返回的数据中有空值！

于是查阅了一些资料怎么对这些进行处理，然后做了一些总结

总结：

  • 空值在控制台打印有两种（null）和<null>
  • nil 表示一个指针指向的对象为空，这个对象的类型是id，显示出来是（null）—>常见于非集合类中
  • [NSNull null] 表示空对象本身，显示出来是<null> ————————–>常见于集合类中
  • NULL 和 nil，没有实质区别，只不过前者是C语言只中的

（null）

在控制台打印（null）有两种

  • nil
  • NULL

nil (空对象)

nil是OC对象的字面空值 ，这个对象的类型是id

NULL

NULL是任意的C指针空值

<null>

  • NSNULL是一个类，它代表空值

  • NSNULL只有一个单例方法 [NSNull null]

  • 用途：用于表示集合值为空的对象(oc中nil是不能存放在集合中的，因为nil是集合结束的标志位)

如何做处理

在开发过程中会遇到很多空的情况，有时候是个空字符串，有时候取得对象（null）,还有时候会得到的情况，那么这几种情况分别到底该如何用BOOL值来判断呢？

针对（null）这种情况，将一个字符串赋值为nil，打印输出结果就是（null）

if(result==nil) { 

    NSLog(@"执行");

}

针对<null>的情况

if([result isEqual：[NSNUll null]]) { 

    NSLog(@"执行");

}

因为我们的后台数据有很多<null>我也懒的找，我的处理方式如下
既然NSNull是一个类，我就写了一个Category添加了一个方法，当使用floatValue有空值得时候就返回 0.0

.h文件

#import <Foundation/Foundation.h>

@interface NSNull (NullCast)

@end

.m文件

#import "NSNull+NullCast.h"

@implementation NSNull (NullCast)

- (float)floatValue
{

    return 0.0;

}

@end
